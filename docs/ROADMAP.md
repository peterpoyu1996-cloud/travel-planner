# ROADMAP

這份文件回答「現在做到哪、接下來做什麼、以後才做什麼」，時間軸/優先順序導向。
跟 [MVP.md](MVP.md) 不同：MVP.md 是「這個版本該不該做這個功能」的範圍清單，
ROADMAP.md 是「這些已同意的功能，大概什麼順序上線」。

## 維護方式

- 每完成一個里程碑（不是每個 commit）才更新這份文件
- 狀態只有三種：✅ 已完成／🚧 進行中／⏳ 未開始，不用寫百分比進度
- 順序調整、優先權改變時要更新，並在 [CHANGELOG.md](../CHANGELOG.md) 留一筆記錄原因

## 資深規劃師缺口評估（2026-07-18 討論，2026-07-18 執行部分項目）

以「這樣的資料夠不夠拼出各式人們需要的行程」為標準做的評估，完整討論見對話紀錄，結論摘要：

**現在的資料拼得出來**：北部沿海景點＋開車＋親子取向的行程（因為 Excel 種子資料剛好是這個取向），加上這次補值後親子適合度標記的覆蓋率大幅提升。

**現在的資料拼不出來（結構性缺口，不是量不夠）**：
1. 不開車的行程（北部/中部完全沒有大眾運輸資料）—— 🚧 部分處理中，見上方「不開車行程地基」
2. 玩水行程安全性資訊 —— 🚧 16/25 筆海灘已查證監視員/設施資料，其餘9筆查無資料
3. 多代同堂/無障礙行程 —— 🚧 找到真實可行的官方資料源（沖縄県観光施設一覧開放資料，含輪椅/電梯/無障礙廁所欄位），但目前只覆蓋16筆知名付費景點，規模還很小，`okinawa-bf-map.jp` 仍然可信度存疑（SSL憑證設定錯）
4. 精算預算（budget_level 目前 11/191 有值，僅限有真實價格根據的連鎖店，其餘沒有票價數字）—— 🚧 部分處理中
5. 季節性資訊（颱風季/水溫/賞鯨季，現在的資料完全沒有時間維度）—— ⏳ 未開始
6. 離島跳島（慶良間諸島等，本來就不在 PRD 範圍內）—— 明確不在近期規劃
7. `rating` 欄位（100% null）—— 明確評估為結構性缺口：A/B級開放來源沒有評分資料，C級（Google/Tripadvisor）禁止爬取，短期內無解，見 data/schema.md
